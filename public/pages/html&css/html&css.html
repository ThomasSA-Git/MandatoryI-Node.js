HTML and CSS

This is simply a walkthrough of some few rules of thumb for HTML and CSS.



Styling

In HTML, there are three methods for managing styling:

  • Inline styles: Styling is applied to a specific element. Can be set with style.
    • Best suited for small projects.
    • No reusability
    • Specificity: Can override styles defined in external stylesheet which can be both a pro and a con.
  • Styles defined within the HTML file: Styling is defined at the top of the HTML document with the style element ul.
    • Best used for specific styling in one HTML document because it can't be shared between different documents.
    • Best suited for smaller projects.
    • Can override external linked stylesheets but be overridden by inline styling.
  • External styles linked via link: Styling is done in a separate CSS-file and the HTML document used the link element to access it.
    • High reusability because it can easily be accessed by multiple HTML documents.
    • Easy refactoring.
    • Helps keep cleaner code and better organization of project.
    • External stylesheets are cached by web browsers which can improve page loading times for returning visitors. Browsers only need to download the stylesheet once and can reuse it for subsequent pages.

Circular dependency

When using scripts in HTML you should be aware of problems that might occur with circular dependency. Circular dependency occurs when two JavaScript files depend on each other's functionality, creating issues with code maintainability and execution.
This can be avoided by refactoring code to break the circular dependency by combining, modularizing or using event-based communication between scripts. Make sure to list the scripts in correct order in the HTML document.
